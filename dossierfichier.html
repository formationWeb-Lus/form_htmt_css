Activité S01 : Travailler avec les fichiers et dossiers

Présentation

Il est essentiel pour les développeurs de logiciels de comprendre en profondeur les fichiers, dossiers et chemins d'accès. Dans ce cours, vous utiliserez de nombreux fichiers pour créer un site Web unique. Il est important de maîtriser leur utilisation et de connaître leurs relations.

Cette activité permettra de revoir les notions essentielles des chemins de fichiers et de définir les conventions de nommage que vous devrez suivre pour ce cours.

Notions essentielles sur les chemins de fichiers

Un chemin de fichier indique comment localiser un fichier dans un système de fichiers. Votre site Web utilise ces chemins pour lier des fichiers tels que les images et les feuilles de style. Si un fichier lié, comme une image, ne s'affiche pas localement ou lors de la publication sur GitHub Pages, vérifiez le fichier et le chemin.

Les termes répertoire et dossier ont essentiellement la même signification. Vous verrez les deux termes utilisés dans les cours WDD et dans l'industrie.

Exemples courants de chemins de fichiers

Chemins absolus

Un chemin absolu donne le chemin complet vers un fichier et est utilisé pour les ressources externes. Il doit commencer par le protocole, comme https://. Les chemins absolus sont identiques, quel que soit l'emplacement du fichier source.

Exemple :

<img src="https://resource.thesite.com/images/breakfast.svg" alt="Exemple de petit-déjeuner">
Chemins relatifs

Un chemin relatif indique comment trouver un fichier par rapport au fichier source. Il peut se trouver dans le même dossier, un dossier parent ou un sous-dossier.

Pour un fichier dans le même dossier : autre.html ou ./autre.html. Pour un fichier dans le dossier parent : ../autre.html. Pour un fichier dans deux sous-dossiers : .html.

Chemin relatif : fichier dans le même dossier

Exemple : apropos-de-nous.html dans le même dossier que le fichier HTML source.

<a href="apropos-de-nous.html">À propos<a>
Chemin relatif : fichier dans un sous-dossier

Exemple : image profile.png dans le sous-dossier images.

<img src="images/profile.png" alt="Xavier Rodriguez" width="100" height="200">
Chemin relatif : fichier dans un dossier parent

Exemple : fichier dans semaine2 qui référence castle.jpg dans le dossier images à la racine.

<img src="../images/castle.jpg" alt="Château en Allemagne" width="300">

Résumé des chemins

  • / : commencer depuis le dossier racine
  • ../ : remonter d’un dossier
  • ../../ : remonter de deux dossiers
  • ./ : commencer dans le dossier courant
  • Pour descendre dans un sous-dossier, indiquer le nom du sous-dossier suivi de /

Normes de nommage des fichiers et dossiers

Pour garantir un travail cohérent et gérable, il est recommandé de suivre des conventions de nommage.

Règles de nommage
  • Tous les noms en minuscules
  • Pas d'espaces, utiliser des tirets -
  • Pas de caractères spéciaux
  • Noms courts et significatifs (sémantiques)
  • Dossiers standards : styles pour CSS, images pour les images

Instructions pour l'activité

Vérification des conventions

Vérifiez chaque fichier et dossier pour voir s'ils respectent les conventions de nommage :

  1. Apropos.html → ❌ Doit être about.html
  2. info d'entreprise.html → ❌ Doit être info-d'entreprise.html
  3. catalogue-deproduits.jpg → ✅ Correct
  4. rendezvous_detrain.html → ❌ Doit être rendezvous-detain.html
  5. produit4.png → ✅ Correct
  6. temple-large (copy).webp → ❌ Doit être temple-large-copy.webp

Exercice sur les chemins de fichiers
